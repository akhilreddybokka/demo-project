Branching in Git.
-------------

1. Head is a pointer for current branch.
2. git branch bug-fix - to create new branch for the copy of current branch we are in , 
    basically head point branch, but it doesnt switch to that branch after creating,
     we are still in the earlier branch.
3. git branch - to list all branches.
4. switch branches - git switch bug-fix
5. git switch -c patch -- to create branch and switch to new branch
6. git diff main new-feature  - to differentiate the changes new branch and main branch.
7. git merge new-feature -- to merge new branch changes to main branch.
8. git push --set-upstream origin new-feature - to merge new branch with new additions 
            to existing branch we use this command.

gir reset
--------------

1. git reset --soft <commit> -- keeps our changes in local directory but removes from stages
2. git reset --mixed <commit> -- keeps our changes in local but removes from committed.
3. git reset --hard <commit> -- removes from both stages and local directory.
